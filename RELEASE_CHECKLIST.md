# DashDrop 发布前清单（Release Checklist）

本文档面向“可发布软件”标准，按优先级拆分为 `P0 / P1 / P2`，可直接作为后续迭代与验收基线。

## P0（发布阻断项，必须完成）

### 1. 可靠性与回归防线
- [ ] 补齐后端关键集成测试（**未实现：需补充对应测试用例**）：
  - [ ] 设备发现上下线（含 `device_lost` 事件）
  - [ ] 发起侧证书指纹强绑定（错误证书必须拒绝）
  - [ ] 多文件并发传输（100 次以上压力回归）
  - [ ] Cancel 语义（已确认保留、未确认删除）
- [ ] 补齐前端 E2E（Playwright）最小闭环（**未实现：无 e2e 测试代码**）：
  - [ ] 设备出现/消失
  - [ ] 发起传输与进度
  - [ ] 完成/失败/取消状态
  - [ ] `identity_mismatch` 告警展示

### 2. 安全底线
- [ ] 验证三平台密钥存储行为（**部分实现**）：
  - [x] macOS Keychain（**已实现**）
  - [x] Windows Credential/DPAPI 路径（**已实现**）
  - [ ] Linux Secret Service（含不可用时的明确错误提示）（**未实现前端错误提示**）
- [ ] 明确并实现“安全存储不可用”时的策略（阻止启动/降级提示/只读模式）（**未实现：无降级与UI提示机制**）。
- [ ] 建立安全事件审计最小集（至少记录 `identity_mismatch` 与关键握手失败）（**未实现**）。

### 3. 传输一致性
- [ ] 校验所有失败事件 payload 统一为：`transfer_id + reason + phase`（**未实现：当前 payload 未完全规范化**）。
- [x] 确保取消路径不会残留临时文件（`*.dashdrop.part` 清理完整）（**已实现：见 `receiver.rs` 的 abort 清理逻辑**）。
- [ ] 校验 mDNS 改名重注册失败时 UI 可见、且配置回滚一致（**未实现：UI 提示不可见**）。

### 4. 工程门禁
- [ ] CI 增加强制门禁（**未实现：缺少 `.github/workflows` 等自动化配置**）：
  - [ ] `cargo check`
  - [ ] `cargo test`
  - [ ] `npm run build`
  - [ ] 集成测试与 E2E（可分 job）

## P1（高优先级，建议首个小版本完成）

### 1. 产品功能完整度
- [ ] 传输任务管理增强：重试、批量取消、失败项重传（**未实现：前端缺少相应管理操作**）。
- [ ] 传输历史页：按设备/时间筛选、失败原因可追踪（**未实现：前端缺少历史页面**）。
- [ ] 配对管理增强：备注、撤销确认、最近使用时间（**未实现**）。

### 2. 传输能力优化
- [ ] 限速与资源策略（前台/后台、按网络条件）（**未实现**）。
- [ ] 文件冲突策略可配置（覆盖/重命名/跳过）（**未实现：无相关配置与交互**）。
- [ ] 大文件与海量小文件性能优化（队列/缓冲参数调优）（**未实现**）。

### 3. 可观测性
- [ ] 结构化日志字段标准化（transfer_id、peer_fp、phase、error_code）（**未实现：仍有部分日志未包含标准维度信息**）。
- [ ] 关键指标采集：成功率、平均耗时、失败分布、取消率（**未实现**）。

## P2（中优先级，体验与可维护性优化）

### 1. 用户体验
- [ ] 首次使用引导（权限、配对、安全说明）（**未实现：缺少 Onboarding 页面或组件**）。
- [ ] 设置页增强（网络状态、服务状态、重注册状态反馈）（**未实现**）。
- [ ] 错误提示可操作化（给出下一步建议）（**未实现**）。

### 2. 架构与长期维护
- [ ] 持久化方案统一评估（JSON vs tauri store vs sqlite）并固化（**未实现：当前 `persistence.rs` 仍为手动 fs 写入**）。
- [ ] 协议演进策略文档（版本兼容、弃用策略、迁移策略）（**未实现**）。
- [ ] 发布说明模板与升级迁移说明模板（**未实现**）。

## 建议执行顺序

1. P0 测试与 CI 门禁  
2. P0 安全与一致性收口  
3. P1 功能增强与性能优化  
4. P2 体验与维护性提升

## 发布验收标准（Definition of Done）

- [ ] 所有 P0 项打勾  
- [ ] CI 全绿且可重复  
- [ ] 至少一轮多设备实机冒烟通过（发现、传输、取消、异常）  
- [ ] 已产出发布说明（已知限制与风险透明）
